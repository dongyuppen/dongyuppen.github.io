## 💡 프로젝트 개요
QR코드를 기반으로 작동하는 사전 진료 연동 시스템입니다. 환자가 개인 스마트폰으로 상세 증상을 기록하고, 생성된 가상 QR 코드를 통해 병원 원무 시스템에 문진 데이터를 즉시 연동할 수 있도록 기획되었습니다. 기존 수기 문진표의 비효율성을 개선하고 데이터 누락을 방지하는 것을 목표로 개발했습니다.

## 🛠 사용 기술
*   **Backend:** Java 17, Spring Boot 3.x, Spring Data JPA
*   **Database:** MySQL 8.0
*   **Infrastructure:** AWS (EC2, RDS, S3)

## 📌 핵심 기여도
*   환자의 사전 문진 내역을 생성, 조회, 수정하는 RESTful API 설계 및 구현
*   병원정보시스템(HIS)과의 데이터 상호운용성을 고려하여 의료 데이터 표준(HL7 FHIR) 규격에 맞춘 JSON 데이터 매핑 로직 구현
*   가상 QR 스캐너 동작 처리를 위한 데이터베이스 모델링 및 인덱스 설계

## 🔥 트러블슈팅 및 성능 개선
**1. QR 코드 생성 및 조회 시나리오에서의 응답 속도 지연 해결**
*   **문제:** 다수의 환자가 동시에 문진표를 제출하고 QR 코드를 생성할 때, 병원 정보와 문진 기록을 JOIN 하는 과정에서 응답 속도가 1.5초 이상 지연되는 병목 현상 발생.
*   **원인:** 문진 기록 테이블에 환자 ID와 생성 일시에 대한 복합 인덱스(Composite Index)가 누락되어 Full Table Scan이 발생함.
*   **해결:** `patient_id`와 `created_at` 컬럼을 묶어 복합 인덱스를 설정하고, JPA의 Fetch Join을 활용해 N+1 문제를 해결함.
*   **결과:** 쿼리 실행 계획(Explain)을 분석하여 Index Scan으로 유도한 결과, 데이터 조회 및 QR 생성 응답 속도를 평균 200ms 이내로 단축함.

**2. 환자 민감 정보 암호화 적용**
*   **문제:** 의료 데이터 특성상 주민등록번호, 연락처 등 민감한 개인정보가 DB에 평문으로 저장될 경우 보안 위협 발생.
*   **해결:** Spring Security의 `BCryptPasswordEncoder` 및 양방향 암호화 알고리즘(AES-256)을 적용하여 DB 저장 시 암호화 처리.
