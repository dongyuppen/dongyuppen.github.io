## 💡 프로젝트 개요
2026학년도 신/편입생 '드림데이즈' 행사를 위해 기획된 실사용 타겟 매칭 서비스입니다. 사용자가 사이트에 접속해 프로필 사진을 등록하고 관심사 정보를 입력하면, 공통점이 많은 새로운 친구를 매칭하여 안내해 줍니다. 

## 🛠 사용 기술
*   **Backend:** Java, Spring Boot, Spring Data JPA
*   **Database:** MySQL
*   **Infrastructure:** AWS (EC2, RDS, S3)

## 📌 핵심 기여도
*   실제 학생들을 대상으로 하는 서비스 백엔드 로직 및 관심사 기반 매칭 알고리즘 쿼리 설계
*   AWS S3를 연동한 프로필 사진 업로드 기능 및 URL 반환 로직 구현
*   AWS EC2와 RDS를 활용한 실서비스 프로덕션 환경 구축 및 서버 모니터링

## 🔥 트러블슈팅 및 성능 개선
**1. 행사 당일 트래픽 스파이크(Traffic Spike) 대응을 위한 커넥션 튜닝**
*   **문제:** 오프라인 행사 당일, 특정 시간대에 신입생 수백 명이 동시에 접속하여 매칭을 요청하자 서버 응답이 지연되고 일부 요청이 Timeout(504 에러) 처리됨.
*   **원인:** HikariCP(커넥션 풀)의 기본 설정(Maximum Pool Size 10) 한계를 초과하여, 다수의 쓰레드가 DB 커넥션을 얻기 위해 대기(Wait) 상태에 빠짐.
*   **해결:** AWS EC2 인스턴스의 메모리 가용량을 계산하여 HikariCP의 `maximum-pool-size`를 30으로 늘리고, `connection-timeout`을 최적화함. 또한 무거운 매칭 로직 쿼리를 단순화함.
*   **결과:** 병목을 해소하여 트래픽 피크 시간대에도 중단 없이 성공적으로 서비스를 제공함.
